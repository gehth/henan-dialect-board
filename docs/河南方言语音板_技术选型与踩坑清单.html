⚠️ 第四步：常见踩坑（必看！）

🔴 致命

💣 坑 1：API Key 写在前端代码里

你用单文件 HTML 时，最容易的做法是 JS 里直接写 APPID/APIKey，但浏览器打开后任何人按 F12 都能看到你的密钥，然后盗用你的额度。更糟的是讯飞/DeepSeek 的 Key 被盗后可能被刷爆产生账单。

✅ 解决：搭一个极简 Python 后端（Flask 10 行代码）当代理。前端只向后端发请求，后端带 Key 调外部 API。或者用 Cloudflare Worker / Vercel Serverless / 阿里云函数计算做无服务器代理。
🔴 致命

💣 坑 2：音频格式不规范导致识别失败 / 乱码

讯飞要求音频：采样率 16kHz，位深 16bit，单声道，编码 PCM (WAV)。浏览器 getUserMedia 默认录出来的可能是 48kHz、立体声或不支持的格式——直接发过去会识别出完全不对的东西，甚至报错。

✅ 解决：在 JS 端录完音频后，用 AudioContext 重采样到 16kHz/单声道，导出 WAV；或在 Python 后端用 FFmpeg 自动转码。
FFmpeg 标准命令：ffmpeg -i input.wav -ar 16000 -ac 1 -acodec pcm_s16le output.wav
🔴 致命

💣 坑 3：讯飞 WebSocket 鉴权签名算错

讯飞语音听写用 WebSocket 连接，需要你把 APIKey + APISecret + 时间戳按 HMAC-SHA256 算法算出一个签名（authorization 参数）。这个签名字符串拼接格式、base64 编码、日期格式差一个字符就连不上——报 401/400 且错误信息很模糊，新手常在这卡半天。

✅ 解决：讯飞官方提供 Python/Java/JS demo，直接用官方 demo 改，不要自己从头写 signature。官网文档目录 → SDK下载 → 语音听写流式版 → Python demo 即可下载。
如果报错 10110 / 10113 / 11200，先检查：① APIKey/APISecret 是否正确复制（注意空格）；② 系统时间是否准确（偏差 > 5 分钟会失败）。
🟡 中等

💣 坑 4：方言识别结果不如想象中准

讯飞/百度确实支持河南话方言识别，但准确率≠100%，尤其是：① 生僻/老旧方言词（如"夜儿个""得劲儿"）可能识别成相近普通话；② 口音太重的长辈录音效果偏差；③ 混合中英文的句子可能分裂。

✅ 解决：① 用 LLM 做二次纠错（把 ASR 结果发给大模型让"修正为正确的方言字"）；② 界面标注"仅供参考"降低预期；③ 长期积累自己的方言词库 JSON 做热词修正（讯飞支持上传热词表）。
🟡 中等

💣 坑 5：大模型"幻觉"编造方言释义

LLM 会自信满满地把一个方言词解释得头头是道，但可能是瞎编的。比如"得劲儿"它有正确说法，但生僻词如"枯楚"（河南话"皱"的意思）可能会胡说。

✅ 解决：① 提示词加一句"如果你不确定，请如实说'不确定'"；② 在界面上标明"释义由 AI 生成，仅供参考"；③ 长期方案：人工校对 + 沉淀可信词库。
🟡 中等

💣 坑 6：浏览器 getUserMedia 录音限制

第一次打开页面，浏览器会弹"是否允许使用麦克风"权限框——如果用户点"禁止"或没注意到，录音功能直接废掉，没有任何明确报错。移动端部分浏览器还不支持 getUserMedia。

✅ 解决：① 页面加明显的"请允许麦克风权限"引导卡片；② 检测权限状态（navigator.permissions.query({name:'microphone'})）；③ 同时支持"上传音频文件"作为备选方案（不依赖麦克风）。
🟡 中等

💣 坑 7：讯飞每日 500 次用完就停

调试阶段你可能会反复上传同一条录音来测试——一次上传、多次调用 ASR API，500 次额度其实不禁用。超量后直接返回错误（错误码 11200+），没有任何降级。

✅ 解决：① 前端做防重复：同一条录音结果缓存到 localStorage，不重复调 API；② 后端加日志记调用次数，快用完时预警；③ 注册时绑定支付宝/微信实名可申请更多免费额度。

🧭 第五步：技术决策树

搭后端代理时，按你的条件走哪条路：

❓ 你会 Python 吗？ → 会 → Flask 后端最简单
├─ 只想用纯前端？ → 可以但必须用 Cloudflare Worker / 腾讯云函数等 Serverless 做代理，不能前端硬编码 Key
├─ 要本地跑还是部署到公网？ → 本地：Flask 跑 localhost → 部署：推荐 Vercel（免费、省事）
├─ FFmpeg 没装？ → 装一个：winget install ffmpeg 或官网下载扔进 PATH
└─ 大模型选哪家？ → DeepSeek（最便宜+中文好）或通义千问（有免费额度）

✅ 第六步：启动前检查表（对照打勾）

  • 讯飞开放平台已注册、已实名、已创建应用并拿到 APPID / APIKey / APISecret
  • 大模型 API（DeepSeek/通义千问）已注册并拿到 API Key
  • 已安装 flask 和 requests（pip install flask requests）
  • FFmpeg 已安装并在终端可用（ffmpeg --version 不报错）
  • 已下载讯飞官方语音听写 Python demo（官网 → 文档 → SDK下载）
  • 已跑通讯飞 demo 的"一句话识别"（确保鉴权成功）
  • 已跑通大模型 API 的"Hello World"（确保 Key 有效）
  • 前端录音方案已确定（getUserMedia + AudioContext 重采样 / 或上传文件）
  • API Key 存储方案已确定（放在后端 .env 或环境变量，不提交到 GitHub）
