📌 一句话定位

用户用浏览器/手机录一段河南话，系统自动：转写成方言字 → 给出普通话对照 + 拼音 → 用大模型解释词义/用法/出处 → 生成一张可分享的"方言卡片"。既能保护濒危方言，又是极好的 AI 技术作品。

② 核心功能模块

🎙
录音 / 上传 浏览器 getUserMedia 录音或传音频文件；支持"长辈模式"大按钮。

🔁

📖

④ 两条技术路线（实测可行）

🅰 云端 API 路线（推荐先做 MVP）

最快出活，难度低，适合一周内跑通主流程。
▸
ASR：讯飞语音听写 / 百度短语音识别 均支持河南话等方言。讯飞开放平台注册→建应用拿 APPID/APIKey，按文档传音频拿结果。
▸
释义/对照/卡片文案：大模型 API 通义千问、DeepSeek、智谱等，把方言语句+指令发过去，返回普通话、拼音、释义、卡片文案。
▸
前端：单文件 HTML 录音→调后端代理（保护 key）→渲染结果→生成卡片图（Canvas）。

⚠️ API Key 不能直接写在前端（会泄露）。学生 demo 可先放本地一个小后端/Serverless 函数做代理，或仅在本地测试时用。

🅱 本地离线路线（进阶亮点）

开源、零上传、还能自己训方言模型——简历上很能打。
▸
ASR：FunASR（阿里达摩院开源） 工业级、支持方言，pip install funasr 即可跑；社区已有粤语/客家话微调模型，可参考自建河南话模型。
▸
释义：本地小模型 / 本地大模型 如 Ollama 跑 qwen2.5 等，全程离线。
▸
价值 "隐私零上传 + 可训练方言识别模型"是两个强卖点，适合当毕业设计/比赛作品。

⚠️ 本地部署需一定算力（CPU 可跑小模型，GPU 更快）；自己训练方言模型要准备方言音频数据，工作量更大。

⑥ MVP 实施步骤（手把手）

  1. 注册 ASR：讯飞/百度开放平台建应用，拿到 APPID/APIKey，跑通官方"语音识别"示例。
  2. 前端录音：单文件 HTML 用 getUserMedia 录 5 秒音频，导出 .wav/.pcm。
  3. 接 ASR：前端把音频发到你的小后端/代理，代理带 key 调讯飞，返回方言文字。
  4. 接 LLM：把方言文字 + 提示词（"翻译成普通话、标拼音、解释生僻词"）发给大模型，解析返回。
  5. 渲染结果：页面分栏展示 方言 / 普通话 / 拼音 / 释义，先能看。
  6. 生成卡片：用 Canvas 把以上内容拼成一张长图，支持"保存图片"。
  7. 打磨 & 开源：加大按钮"长辈模式"、补 README、GitHub 发布，进项目列表。

⑧ 难点与对策

难点对策
方言 ASR 准确率有限选已支持河南话的厂商 API；结果经 LLM 二次纠错；标注"仅供参考"。
生僻词 LLM 会编输出加"需人工校对"；沉淀可信词库覆盖高频词。
API Key 安全经后端/Serverless 代理调用，不在前端硬编码。
方言写法不统一以"音近字+拼音+释义"为主，不强行规范汉字。
语料不足先自己录 20–50 条跑通闭环，再众包扩展。
